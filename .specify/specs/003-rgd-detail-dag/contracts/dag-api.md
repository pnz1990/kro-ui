## 1. Pure Function: `buildDAGGraph`

**File**: `web/src/lib/dag.ts`
**Signature**:

```typescript
export function buildDAGGraph(spec: Record<string, unknown>): DAGGraph
```

**Input**: The `spec` property of a kro RGD object (i.e., `rgd.spec`).

**Output**: A `DAGGraph` containing all nodes with layout coordinates, all
dependency edges, and total graph dimensions.

**Behavior**:
1. Extracts root node from `spec.schema` (always present, always ID `'schema'`)
2. Iterates `spec.resources[]` (may be `null`, `undefined`, or empty array)
3. Classifies each resource into one of 5 `NodeType` values
4. Extracts CEL expressions from templates via deep object traversal
5. Builds edges from CEL references between resources
6. Ensures all nodes are reachable from root (implicit edges for orphans)
7. Runs BFS-layered layout algorithm
8. Returns graph with all coordinates assigned

**Invariants**:
- Root node (id `'schema'`, nodeType `'instance'`) is always present
- `nodes.length >= 1` (at minimum, the root)
- All edges reference valid node IDs
- Layout is deterministic: same input Ôćĺ same output
- `specPatch` is never produced as a node type (FR-012)

**Error handling**:
- `spec.resources` is null/undefined Ôćĺ return graph with root node only
- Resource has neither `template` nor `externalRef` Ôćĺ classify as `'resource'`
  with warning (empty template)
- Invalid CEL expression (unclosed `${`) Ôćĺ skip silently

## 2. Component: `DAGGraph`

**File**: `web/src/components/DAGGraph.tsx`
**Props**:

```typescript
interface DAGGraphProps {
  graph: DAGGraph
  onNodeClick?: (nodeId: string) => void
  selectedNodeId?: string
}
```

**Behavior**:
- Renders inline `<svg>` with `data-testid="dag-svg"`
- Dimensions: `width` and `height` from `graph`
- For each node: `<g data-testid="dag-node-{id}">` containing:
  - `<rect>` with type-specific CSS class
  - `<text>` with node label
  - Type badge (`ÔłÇ`, `?`, `ÔČí`) as appropriate
- For each edge: `<path>` with cubic bezier curve and arrowhead marker
- Selected node gets highlight border
- Nodes are `tabindex="0"` with `role="button"` and keyboard Enter to select
- No kro-specific logic ÔÇö purely renders the DAGGraph data structure

**CSS classes** (applied to node `<g>` elements):
- `.dag-node` ÔÇö base class for all nodes
- `.dag-node--instance` ÔÇö root node
- `.dag-node--resource` ÔÇö managed resource
- `.dag-node--collection` ÔÇö forEach collection
- `.dag-node--external` ÔÇö external reference (single)
- `.dag-node--externalCollection` ÔÇö external reference (collection)
- `.node-conditional` ÔÇö modifier for includeWhen nodes (any base type)
- `.dag-node--selected` ÔÇö currently selected node

**Data-testid mapping**:
- `dag-svg` ÔÇö the root `<svg>` element
- `dag-node-{id}` ÔÇö each node group (e.g., `dag-node-schema`, `dag-node-appNamespace`)

## 3. Component: `NodeDetailPanel`

**File**: `web/src/components/NodeDetailPanel.tsx`
**Props**:

```typescript
interface NodeDetailPanelProps {
  node: DAGNode
  onClose: () => void
}
```

**Behavior**:
- Fixed/absolute-positioned panel on the right side, width 400px
- Shows node details:
  - Kind badge: `<span data-testid="node-detail-kind">{node.kind}</span>`
  - Node type concept explanation: `<div data-testid="node-detail-concept">`
  - CEL expressions via `<KroCodeBlock>` (readyWhen, includeWhen, forEach)
  - For root node: SimpleSchema spec fields, status projections
  - For external nodes: referenced apiVersion/kind/name
- Close button: `<button data-testid="node-detail-close">ÔťĽ</button>`
- Panel container: `<div data-testid="node-detail-panel">`

**Concept text per node type**:

| NodeType | Concept text |
|----------|-------------|
| `instance` | "Root Custom Resource ÔÇö The CR generated by this RGD. Its spec fields are defined by the schema, and its status is projected from child resource values." |
| `resource` | "Managed Resource ÔÇö A Kubernetes resource created and owned by kro when an instance of this RGD exists." |
| `collection` | "forEach Collection ÔÇö A set of Kubernetes resources created by iterating over a CEL expression. Each item produces one resource instance." |
| `external` | "External Reference ÔÇö A reference to a pre-existing Kubernetes resource. kro reads it but does not create or own it." |
| `externalCollection` | "External Reference Collection ÔÇö References to pre-existing Kubernetes resources matched by label selector. kro reads them but does not create or own them." |

## 4. Page: `RGDDetail`

**File**: `web/src/pages/RGDDetail.tsx`

**URL**: `/rgds/:name`
**Query params**: `?tab=graph|instances|yaml` (default: `graph`)

**Behavior**:
1. Fetches RGD via `getRGD(name)` on mount
2. Shows loading/error states
3. Renders tab bar: Graph, Instances, YAML
4. Graph tab: calls `buildDAGGraph(rgd.spec)`, renders `<DAGGraph>` with
   `onNodeClick` wiring, conditionally renders `<NodeDetailPanel>` when a node
   is selected
5. Instances tab: placeholder for spec 004
6. YAML tab: renders `<KroCodeBlock code={toYaml(rgd)} />`
7. Active tab synced to URL query parameter

**Data-testid mapping**:
- `tab-graph` ÔÇö Graph tab button
- `tab-instances` ÔÇö Instances tab button
- `tab-yaml` ÔÇö YAML tab button
